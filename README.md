# StockPulse
Self-hosted приложение для просмотра и аналитики чужих сделок и портфелей в соцсети Пульс

### Текущая задумка

Приложение, которое позволяет через API-ключ Тинькофф-инвестиций обращаться к списку подписок в Пульсе.
Приложение собирает данные публичных портфелей(сделки, объем портфеля, доля компаний в портфеле) для последующего анализа и сравнения в разных выборках.
Все это нужно, чтобы принимать решения по торгам и стратегиям(ориентировочно в облигациях)

### Стек

- FastAPI 
- Docker + docker-compose или k8s
- Nginx
- Postgres/SQLite (для хранения подписок и активностей)
- Тинькофф API (gRPC)
- Grafana
- Prometheus

Выбор обусловлен требованиями проекта и моим личным желанием поиграться с новыми технологиями на практике.

### Запуск проекта

#### Создать и заполнить .env файл
```shell
cp .env.example .env
```

#### Запустить сетап
Команда создает init файл для БД и datasource для grafana
```shell
make app_setup
```

#### Docker-compose
```shell
docker-compose up --build
```

UPD 20.08.2025

Проведя исследование возможностей API Т-банка, выяснилось, что нет возможности работать Пульсом напрямую. 
Из-за этого решено ~~завершить проект~~ собрать список эндпоинтов через DevTools из веб клиента Пульса, чтобы написать собственное решение для парсинга необходимых данных.
На данный момент основное препядствие - авторизация в сети Пульс. Интересующая меня информация защищена авторизацией. Так как я хочу сделать полностью автоматизированный инструмент, мне нужно как-то авторизовываться в сети Пульс. 
Так как полноценного API для этого нет, решено использовать Selenium для получения access и refresh токенов для последующего использования в HTTP запросах. 
Из сложностей - требование введения временного кода, который приходит на номер телефона. На данный момент я не знаю, как это автоматизировать. В голову приходит только реализация через MacOS Shortcut, но это решение требует ресерча.
